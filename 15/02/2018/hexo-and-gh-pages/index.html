Cómo usar Hexo para crear un blog y alojarlo en github-pages

Apuntes de un unicornio
15-02-2018
Hexo, gh-pages, tutoriales

¿Qué es Hexo?

Hexo es un framework estático para hacer blogs desarrollado en node.js. De los 3 frameworks de blogs que he probado (Jekyll, Hugo y Hexo) este es con diferencia para mí, el más sencillo de usar e implementar en github pages.
Hexo te permite escribir los posts en markdown y despues generar el sitio para su publicación en segundos utilizando un theme para los estilos.
Documentación oficial.
Repositorio en Github.

La carpeta assets

La carpeta global “source” es donde guardaremos cualquier anexo que necesitemos para nuestro blog, como archivos css, js, imágenes, pdfs…
Por ejemplo, lo más común en un blog es tener imágenes que queremos insertar en nuestros post, por lo que podríamos crear una carpeta “images” dentro de sources para guardarlas y después acceder a ellas a través de la ruta “images/image.png”.
También hay una forma de asignar una carpeta en assets por cada post por si queremos organizar los anexos de una forma más metódica, para ello sólo debemos habilitar la organización por carpetas de posts, aunque si lo hacemos no podremos linkar las imágenes usando markdown sino que tendremos que usar código específico de Hexo.
