Master class sobre SQL

Apuntes de un unicornio
10-05-2018
MySQL-workbench, mysql, tutoriales

Estos apuntes están pensados para trabajar en Debian 9 con Mariadb.

MySQL Workbench

MySQL-workbench es un paquete que podemos instalar por la terminal usando $ sudo apt install mysql-workbench.
MySQL workbench sirve para diseñar implementar y modificar nuestras bases de datos a través de una interfaz gráfica. También nos sirve para manipular los datos ya ingresados en las BBDD. Podemos ver los rendimientos y los cuellos de botella…

Creación de una base de datos

Para crear una base de datos en mysqlWorkbench lo primero que necesitamos es crear una conexión, haciendo click en el botón más podemos nombrarla, elegir el servidor y el usuario con el que ingresar. Si tiene contraseña nos la pedirá.
A partir de aquí podemos crear una nueva base de datos haciendo click en el cuarto icono de la barra de iconos de bajo de la barra de los menús, que representa a una BBDD con un +. Nos pedirá escoger un nombre y un cotejamiento (latin-swedish-ci). Nos confirma el comando que va a ejecutar:

CREATE SCHEMA `sql_course_wb` ;

Creación de una tabla dentro de una base de datos

Para crear una tabla dentro de una base de datos, desplegamos el icono triangular a la izquierda del nombre de la base de datos a la que queremos añadirle la tabla, hacemos click derecho en tables y seleccionamos create new table. Aparecerá una ventana en la que elegir el nombre y un menú donde por columnas crear los campos con su número y marcando sus propiedades con checkboxes.

Borrar un usuario DROP USER '[nombre]'@'localhost'; Borra un usuario de mysqli
Concede permisos a un usuario GRANT [permiso] ON * . * TO '[usuario]'@'localhost'; Concede permisos a un usuario en bbdd.tablas. Los asteriscos son comodines que al usarlos significan TODO.
Revocar permisos a un usuario `REVOKE [permiso] ON [nombreBBDD].[nombreTabla] FROM ‘[usuario]’@’localhost’ Revocar permisos a un usuario

Mostrar la versión SELECT version(); Ver la versión actual de mysql

Listado de propiedades de campos

Propiedad descripción
NOT_NULL El campo no es nulo

Ejemplo de creación de una tabla en una base de datos

$ mysql -u root -p
use sql_course;
CREATE TABLE user_type (
id INT NOT NULL AUTO_INCREMENT,
name VARCHAR(20) NOT_NULL,
PRIMARY KEY (id)
)ENGINE=InnoDB DEFAULT CHARSET=latin1;

Hasta que no ponemos ; no se cierra la instrucción. A la hora de crear un campo definimos el nombre y el tipo y después sus propiedades. Separamos la definición de cada campo con una coma. Debemos asignar que campo es el índice de la tabla. Por último hay que especificar el motor que utiliza y el tipo de caracteres.

Leer datos de una tabla

Para leer los datos de una tabla debemos ejecutar select especificando los campos que queremos ver y la tabla de la que los queremos leer. Ejemplo para ver los usuarios en una base de datos de moodle paso a paso: 

mysql -u root -p
use [nombreBBDD];
show tables;
describe [nombreTabla];
select [campo] from [tabla];
